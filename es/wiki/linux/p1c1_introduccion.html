Capítulo 1: Introducción a GNU Linux

¿Qué es Linux realmente?

Linux no nació en el vacío. Sus raíces se remontan a Unix, desarrollado en los laboratorios Bell de AT&T en 1969 por Ken Thompson, Dennis Ritchie y otros. Unix introdujo conceptos revolucionarios: todo es un archivo, sistema jerárquico de directorios, y la filosofía de "hacer una cosa y hacerla bien". Su diseño modular y portabilidad (escrito en C) lo hicieron el sistema operativo preferido en entornos académicos y empresariales.

En 1991, Linus Torvalds, un estudiante finlandés, anunció en un foro que estaba creando un "sistema operativo libre" como hobby. Linux era solo el kernel: el núcleo que gestiona hardware, memoria, procesos y permisos. Lo publicó bajo la licencia GPL, permitiendo a cualquier persona usar, modificar y redistribuir el código. Este gesto democratizó el acceso a un sistema tipo Unix.

Paralelamente, desde 1983, Richard Stallman inició el Proyecto GNU con el objetivo de crear un sistema operativo completamente libre. Para 1991, GNU tenía casi todas las herramientas necesarias (compilador GCC, biblioteca glibc, Bash), pero carecía de un kernel funcional. El kernel de Linux vino a completar ese vacío, dando lugar a sistemas operativos completos.

Hoy en día, el termino Linux se emplea coloquialmente para hacer referencia a todo un sistema operativo, sin embargo, en realidad solo es el kernel (más conocido como núcleo). Lo adecuado sería decir GNULinux para reconocer la aportación de GNU.

¿Qué es el Software Libre?

Software Libre (Free Software) se define por cuatro libertades esenciales:

  • Libertad 0: Usar el programa para cualquier propósito.
  • Libertad 1: Estudiar cómo funciona y adaptarlo a tus necesidades (acceso al código fuente).
  • Libertad 2: Redistribuir copias para ayudar a otros.
  • Libertad 3: Mejorar el programa y publicar las mejoras.

El "free" se refiere a **libertad**, no necesariamente a precio gratuito.

¿Qué es una distro (distribución Linux)?

Una distribución Linux es un sistema operativo completo que empaqueta:

  • El kernel Linux
  • Herramientas GNU (o alternativas)
  • Un sistema de gestión de paquetes
  • Software adicional (entornos de escritorio, aplicaciones)
  • Configuraciones por defecto y soporte

Principales distros

>Debian — Modelo comunitario estable

  • Filosofía: Estabilidad sobre novedad.Software meticulosamente probado.
  • Ciclo: +-2 años. Versiones Stable, Testing, Unstable
  • Gestión: Desarrolladores voluntarios con voto democrático
  • Ética: Compromiso fuerte con el software 100% libre.

    • Red Hat — Modelo empresarial

    • Filosofía: Estabilidad empresarial con soporte comercial.
    • Productos: RHEL (pago con soporte), Fedora (comunitario, cutting-edge), CentOS (estable, gratis).
    • Ciclo: Lanzamientos predecibles con soporte a largo plazo (LTS).
    • Modelo: Open Source con negocio basado en servicios.

    Arch Linux — Modelo rolling release

    • Filosofía: Simplicidad, minimalismo y actualizaciones continuas.
    • Ciclo: Rolling release (sin versiones fijas, actualizaciones constantes).
    • Enfoque: "Keep It Simple", configuración manual, Wiki excelente.
    • Pacman: Gestor de paquetes rápido y sencillo.

    ¿Por qué usaremos Debian como base en este libro?

  • Estabilidad: Ideal para aprender sin rupturas inesperadas.
  • Universalidad: Base de Ubuntu, Mint y muchas otras distros.
  • Documentación: Comunidad enorme y recursos abundantes.
  • Ética: Coherencia con los principios del software libre.
  • Educativo: Configuración más transparente que distros muy automatizadas.
